# 要求
项目目前为一个 kotlin + mc forge 1.20.1 的样板。要求在此样板上按要求制作一个 mc forge 1.20.1 的模组。

具体要求如下：

- 规格为“服务器必装，客户端选装”
- 项目管理使用 gradle-kts
- 包名 `top.tdrgame.auth`
- 确保模组性能足够好，与其它模组（如格雷）兼容完好。

服务器端模组功能：
- 模组为服务器提供对离线玩家的验证
- 模组新增 `\login <PASSWD>` 和 `\register <PASSWD> <PASSWD AGAIN>` ， `\resetpasswd <PLAYER>` （密码重置指令仅 OP 可用）
- 玩家离线进入时，必须登录/注册 。未登录超过一定时间/多次输入密码错误则踢出服务器。
- 玩家未登录、注册时禁止发言、除 `\login` `\register` 外的指令、移动、与物品交互、拾取掉落物。物品栏物品不显示，不会扣血&死亡。
- 当“玩家死亡后直接退出服务器没有重生”后进入时，先自动重生到其出生点再要求登录。
- 当玩家之前使用离线登录，首次正版登录时同样需要 `\login` 。验证后正版登录就永远不需要了。
- 当玩家之前使用正版登录，想要离线登录则必须在正版登录状态下使用 `\register` 命令；在离线模式下用 `\login` 即可登录。
- 当 OP 执行 `\resetpasswd` 后在需要玩家登录的场景下须重新 `\resetpasswd` 。
- 如玩家开始就用正版登录，则正版登录时不需要验证。
- 玩家密码必须使用密码学安全的哈希算法存储。
- 提供如下模组配置：是否启用验证（默认禁用）；登录超时时间（默认 90 秒）；密码错误宽容次数（默认 5 次）；密码加盐配置。
- 支持从 offlineauth (../offlineauth) 和 TrueUUID (../TrueUUID/) 迁移密码等数据（当配置中首次启用验证时尝试进行），确保可以无缝切换。

当客户端也安装该模组时额外提供这些功能：
- 哈希计算移到客户端进行
- 基于 TDLib 显示登录/注册界面
- 当 IP 和机器唯一 ID 不变时支持自动登录


此外，提供 README 和 github ci 。
